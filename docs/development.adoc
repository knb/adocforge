= AdocForge Development Guide

== Requirements

* Node.js 20 or later
* Corepack

== Setup

[source,shell]
----
corepack enable
pnpm install --frozen-lockfile
----

== Checks

[source,shell]
----
pnpm lint
pnpm format:check
pnpm typecheck
pnpm test
pnpm build
----

Run `pnpm test:e2e` for browser-level editor workflows.

Run `pnpm test:a11y` for the focused axe-core WCAG A/AA audit and keyboard workflow. The full E2E command includes these checks.

Run `pnpm pack:packages`, `pnpm verify:consumer`, and `pnpm verify:publish` before a release candidate. These commands must inspect the current `@adocforge/*` packages before Phase 5 is complete.

`pnpm publish:packages` performs a dry-run unless the explicit publish safeguards documented in `docs/releasing.adoc` are satisfied.

== Playground

[source,shell]
----
pnpm --filter @adocforge/playground dev
----

The playground must consume public package APIs. It must not import files from another package's `src` directory.

See `docs/api.adoc` for the supported integration surface. Public API changes must update that reference and the owning package README in the same commit.

== Architecture Changes

Read `ADOCFORGE_BOOTSTRAP.md`, `docs/architecture.adoc`, and relevant ADRs first. Add an ADR for decisions that change package responsibilities, public API direction, trust boundaries, persistence, or external service integration.

== Legacy Migration

Code under `legacy/` is retained as tested migration input. Move behavior into the new packages by responsibility and add tests at the new boundary. Do not make new public APIs depend on legacy package names.
