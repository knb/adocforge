== レイヤの定義

本表の ✓ / △ / — は link:memo-body-editor-roadmap.adoc[メモ本文エディタ ロードマップ] の *Phase 6c 基本方針*（**ソース優先**）に沿って記載する。

* *正（保存・同期）:* DB / フォームに渡す AsciiDoc プレーン文字列。WYSIWYG タブではユニットごとの stored source（`setUnitAdocSource` / `getUnitAdocSource`）。
* *CM / プレビュー:* 見せ方のみ。HTML 逆変換・contentEditable で本文を書き換えない（`AGENTS.md`）。

[cols="1,3"]
|===
|レイヤ |説明

|HL
|`@kbmemo/adoc-codemirror` — CM6 向け AST ハイライト（`refreshHighlights`）。ソースタブ全文 CM・WYSIWYG ユニット内 CM の双方で利用。

|EU
|WYSIWYG ブロック分割（`parseEditUnitsFromSource`）。WYSIWYG タブのユニット境界の正。

|PV
|クライアント Asciidoctor プレビュー（`@kbmemo/adoc-preview`）。主に *ソースタブ* の分割プレビュー。

|WL
|*ソースタブ* — 全文 CM6 + Phase 4–5 Decoration WYSIWYG ライト（`wysiwyg_lite.js`、`wiki_link_wysiwyg.js` 等）。カーソル位置で raw 復帰。Compartment でオンオフ可。

|WY
|*WYSIWYG タブ（既定）* — `@kbmemo/adoc-wysiwyg`。EU で分割した各ユニットは *プレビュー*（読取専用 Asciidoctor HTML）と *編集*（生 AsciiDoc のユニット内 CM、`createKbmemoWysiwygSourceExtensions({ rawSourceMode: true })`）の二状態。クリックでプレビュー → ソース CM。保存は stored source のみ（`unitToAsciidoc` は HTML フォールバックしない）。

|SV
|コミット後表示（サーバー Asciidoctor + KBMemo 前処理、`memo_html`）。
|===

== フェーズ計画（全記法対応）

=== Phase A — フィクスチャと計測（完了）

. `syntax-quick-reference.adoc` 作成（76 `syntax-ref` マーカー）
. Vitest smoke — 全文書 + セクション別 HL / EU、`syntax_coverage_sync`（fixture ↔ roadmap 同期）
. 本ロードマップでギャップ可視化（HL / EU / PV / WL / WY / SV 対応表）

*成果物:* `test/fixtures/asciidoc/`、`packages/adoc-codemirror/test/syntax_*`、`scripts/report-syntax-coverage.mjs`

=== Phase B — ブロック境界の拡張（EU 完了 / WY 第1段）

. 定義リスト・qanda・checklist の edit unit 境界 — `pushDlistUnit` / `dlistEndLine`
. `[NOTE]====` 等 delimited admonition — 区切りブロック + preamble（タイトル行）
. sidebar / example / blockquote / open block — delimited preamble + paragraph quote 復元
. CSV / DSV 表 — `isTableDelimiterLine` に `,===` / `:===` 追加（EU はソース単位）

*テスト:* `packages/adoc-codemirror/test/phase_b_edit_units.test.js`、`packages/adoc-codemirror/test/literal_paragraph.test.js`
*WY（2026-05）:* ユニット内 CM は常に生 AsciiDoc（`rawSourceMode: true`）。literal 段落の Enter 継続・空行ユニット分割、リスト／チェックリスト Enter 継続。プレビューは Asciidoctor HTML（`flattenAndWrapUnits` で quoteblock / sidebarblock / exampleblock 等をユニット化）。表セルの視覚編集は Phase E 以降。

=== Phase C — インラインと HL 拡張（HL 完了）

. unconstrained 強調・`#highlight#`・上付き/下付き — `inlineCapture` / CM テーマ
. xref / anchor / inter-document link の HL 装飾 — `adoc-anchor` / `adoc-xref`
. literal paragraph / hard break の行認識 — `highlightLiteralBlock` / `adoc-hardbreak`

*テスト:* `packages/adoc-codemirror/test/phase_c_highlight.test.js`、`inline_capture.test.js`

=== Phase D — マクロとメディア（HL 完了）

. audio / video / kbd / menu / btn — `highlightMediaMacrosInText` / `inlineCapture` 拡張
. inline image + positioning role — 既存 `highlightImageMacrosInText`（インライン / alt 属性）
. footnote / bibliography — `adoc-footnote` / xref・anchor HL（編集 UX はソース単位維持）

*テスト:* `packages/adoc-codemirror/test/phase_d_highlight.test.js`

=== Phase E — 高度機能（HL / EU 第1段 完了）

. include 行の HL / EU — `adoc-include`、1 行 1 ユニット
. 属性定義・参照・counter — `adoc-attribute` / `adoc-attribute-ref`
. passthrough インライン / ブロック — `adoc-passthrough`、`++++` 区切り EU
. コメントブロック・document header・区切り線 — `////` / author / `'''` / `<<<`
. delimited admonition の区切り HL — `[NOTE]` + `====`

*テスト:* `phase_e_highlight.test.js`、`phase_e_edit_units.test.js`
*WY:* 表セル結合・セル内ブロックの *視覚的* 編集は未着手（テーブルは raw ソース編集 + プレビュー）

=== Phase F — リスト・置換・Markdown 互換 HL（完了）

*テスト:* `phase_f_highlight.test.js`

== 作業ルール

* 各 PR は `syntax-ref` id を 1〜3 個に絞る
* フィクスチャ該当節で手動確認 + 必要なら `packages/*/test/` に id 別テスト追加
* 表示の正は Asciidoctor（SV）。CM は軽量パースでよい
* **WY レイヤ:** EU・Asciidoctor プレビュー・stored source CM 編集で足す。`annotateParagraphAttributes` 等はプレビュー DOM へのメタデータ付与のみ。HTML 逆変換・contentEditable・ユニット CM の Decoration 編集（`rawSourceMode: true`）は編集経路にしない（`AGENTS.md` / Phase 6c 基本方針）
* **WL レイヤ:** ソースタブの Decoration WYSIWYG のみ。WYSIWYG タブのユニット CM には適用しない
* ロードマップ表の ✓/△/— を PR マージ時に更新
