== フェーズ計画（全記法対応）

=== Phase A — フィクスチャと計測（本ブランチ）

* `syntax-quick-reference.adoc` 作成
* Vitest smoke（HL + EU が例外なく通ること）
* 本ロードマップでギャップ可視化

=== Phase B — ブロック境界の拡張（EU / WY 優先）

. 定義リスト・qanda・checklist の edit unit 境界
. `[NOTE]====` 等 delimited admonition
. sidebar / example / blockquote / open block
. CSV / DSV 表（表示は SV 正、EU はソース単位維持で可）

=== Phase C — インラインと HL 拡張

. unconstrained 強調・`#highlight#`・上付き/下付き
. xref / anchor / inter-document link の HL 装飾
. literal paragraph / hard break の行認識

=== Phase D — マクロとメディア

. audio / video / kbd / menu / btn
. inline image + positioning role
. footnote / bibliography（編集 UX はソース維持、PV/SV 優先）

=== Phase E — 高度機能

. include（タグ / 行 / URL）— safe mode 方針要決定
. 属性置換・counter・passthrough の編集補助
. 表セル結合・AsciiDoc セル内ブロックの WYSIWYG 往復

== 作業ルール

* 各 PR は `syntax-ref` id を 1〜3 個に絞る
* フィクスチャ該当節で手動確認 + 必要なら `packages/*/test/` に id 別テスト追加
* 表示の正は Asciidoctor（SV）。CM は軽量パースでよい
* ロードマップ表の ✓/△/— を PR マージ時に更新
